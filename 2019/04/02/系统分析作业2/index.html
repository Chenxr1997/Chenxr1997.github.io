¶简答题

¶用简短的语言给出对分析、设计的理解

分析（analysis）强调的是对问题和需求的调查研究，而不是解决方案，找出构成问题域概念模型的关键抽象（行为和交互）。例如，如果需要一个新的在线交易系统，那么应该如何使用它？他应该具有哪些功能？

设计（design）强调的是满足需求的概念上的解决方案（软件，硬件方面），而不是其实现。在体系结构框架内构建系统。设计过程将分析的抽象结果映射到软件设计类层次结构中，将对象（类实例）及其行为组合到系统协作中，还需要在设计过程中发现和设计一些不在问题域中，但是在实现中需要的抽象，最后在层次结构中组织类。例如，对数据库和软件对象的描述。设计思想通常排斥底层或"显而易见"的细节。

¶用一句话描述面向对象的分析与设计的优势

在面向对象分析（objected-oriented analysis）过程中，强调的是在问题领域内发现和描述对象（或概念）。例如，航班信息系统中的飞机，航班，飞行员等概念。

在面向对象设计（objected-oriented design）过程中，强调的是定义软件对象以及它们如何协作以实现要求。例如，软件对象飞机可以有tailNumber属性和getFlightHistory方法。

优势：面向对象分析与设计根据设计的需要对现实世界的事物进行抽象，产生类。使用这样的方法解决问题，接近于日常生活和自然的思考方式，系统分析师不需要懂专业术语，问题领域和实现层面的专家可以用平常通用的语言沟通，使得分析结果更加符合需求。

¶简述 UML（统一建模语言）的作用。考试考哪些图？

统一建模语言（UML）是描述，构造和文档化系统制品的可视化语言。

作用：

UML是一种进行面向对象程序设计的工具，将OOA（面向对象分析）与OOD（面向对象设计）用统一的符号表示出来，讲对象属性和对象之间的联系可视化，用来把现实中的问题抽象成面向对象的解决方案，以便进一步的编码。具体来说，可以用作有如下作用：

1.作为草图：使用UML设计非正式的，不完整的图，借助可视化语言的功能，用于探讨问题或解决问题方案的复杂部分。

2.作为蓝图：使用UML设计相对详细的设计图，可以用于1）逆向工程，即以UML图的方式对现有代码进行可视化，使其容易理解；2）代码生成（前向工程），使用图生成代码，然后开发者编写并补充其他代码。

3.作为编程语言：用UML完成软件系统可执行规格说明，可执行代码能够被自动生成，但并不像通常一样为开发者所见或修改；人们仅使用UML"编程语言"进行工作。

¶从软件本质的角度，解释软件范围（需求）控制的可行性

软件的本质有：

1.complexity 复杂性
2.conformity 一致性
3.changeability 可变性
4.invisibility 不可视性

由于软件的复杂性和可变性，一个需求的改变就可能引发整个软件设计的改变，因此如果不对软件的需求范围做出控制，会增加开发难度，对软件制品的质量产生负面效果，因此软件范围的控制具有必要性。

又由于软件的不可视性，难以在初次测试时检测是否满足确定真正的需求。所以软件范围的控制具有困难性。

但是在多数情况下，客户与开发者能就项目的 20% 内容给出严格的需求约定，80% 的内容都是相对模糊的。因此，围绕客户目标，发现并满足客户感兴趣的内容是最关键的。因此在需求分析与设计的过程中，对现有的需求进行评估，对需求的重要性进行评级并排序，初次迭代中开发有较大价值的需求，并对软件的增量留有足够的空间，同时围绕软件开发的成本进行调控，这样可以使得软件生产的范围，预算、工期、意外变化等在一定程度都在我们的掌控之中。

¶项目管理实践

¶看板使用练习

看板（kanban）是敏捷开发的重要手段，主要用于项目的进度管理。所有需要完成的任务，都做成卡片，贴在一块白板上面，这就是看板。

按照不同的阶段，看板分成若干列：

  • Todo （待开发）
  • Doing （开发中）
  • Done （已完成）

可以根据实际情况，分出更多的列：

  • Todo （待安排）

  • Plan （计划）

  • Develop （开发）

  • Test （测试）

  • Deploy （部署）

  • Done （已完成）
